# 您好 Hexo，再见 Halo

还记得第一次接触 Hexo 的时候。

我以为：

> “搭建博客，应该是一件很酷的事情。”

于是：

我开始安装 Node.js。

安装 Git。

安装 Hexo。

部署 Vercel。

绑定 Cloudflare。

修改 Butterfly。

添加 Live2D。

加入音乐播放器。

修改字体。

配置导航栏。

折腾了很久很久。

---

# 折腾，才是真正的开始

Hexo 很强。

也很自由。

但自由的代价就是：

> 你需要面对无数 YAML 报错。

可能只是：

一个空格。

一个缩进。

一个重复字段。

整个博客就会瞬间爆炸。

我开始明白：

原来程序员最害怕的不是 Bug。

而是：

```yaml
duplicated mapping key
```

---

# 那些深夜

我一次又一次：

```bash
hexo clean
hexo g
hexo s
```

浏览器一次又一次刷新：

```text
localhost:4000
```

有时候成功。

有时候白屏。

有时候 CSS 全丢。

有时候播放器不见了。

有时候 Live2D 又炸了。

但也正因为这些折腾。

我真正开始理解：

- Linux
- Docker
- Nginx
- Cloudflare
- CDN
- 静态网站
- 前端结构

---

# Hexo 教会了我很多

虽然它很折腾。

但它也让我第一次真正拥有：

属于自己的博客。

那种感觉很奇妙。

像是在互联网世界里。

拥有了自己的一个小房间。

---

# 后来，我遇见了 Halo

比起 Hexo。

Halo 更像：

```text
真正能长期写文章的平台
```

不需要反复改 YAML。

不需要到处找插件。

不需要担心：

```text
hexo g 又报错
```

它更像：

> “专注写作，而不是专注修 Bug。”

---

# 所以

不是 Hexo 不好。

只是：

现在的我。

更想把时间花在：

- 写文章
- 分享生活
- 记录成长

而不是：

```bash
npm install
```

---

# 您好 Hexo

谢谢你让我入门。

让我第一次真正理解：

什么叫静态博客。
